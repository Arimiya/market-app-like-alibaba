# Messages Page

## Purpose

The messages page supports customer and vendor conversations, especially around orders and product questions.

## Primary Users

- Customers.
- Vendors.
- Support admins.

## Key Sections

- Conversation list.
- Chat panel.
- Message composer.
- Attachment upload.
- Order-linked context panel.

## Core Requirements

- Support customer-to-vendor chat.
- Support text, images, and file sharing.
- Link conversations to orders when relevant.
- Allow admins to review messages only for support or dispute workflows.
- Store message timestamps and read status.

## MVP Data Needed

- Conversation.
- Participants.
- Messages.
- Attachments.
- Linked order.
- Read status.
