# Notifications

## Purpose

Notifications keep customers, vendors, and admins informed about marketplace events.

## Channels

- Email.
- SMS.
- Push notifications.
- WhatsApp.

## Customer Events

- Order placed.
- Payment received.
- Shipment update.
- Delivery completed.
- Refund processed.
- Wishlist item discounted.

## Vendor Events

- New order.
- Payment confirmed.
- Low stock alert.
- Return request.
- Payout processed.
- New customer message.

## Admin Events

- Vendor approval request.
- Dispute opened.
- Refund request.
- Fraud flag.
- High-risk transaction.

## MVP Requirements

- Store notification preferences.
- Send transactional messages reliably.
- Keep an in-app notification list.
- Support retry handling for failed sends.
